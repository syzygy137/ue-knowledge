# RTX 5090 D3D12 Crashes in UE5

## Symptoms
- NvPresent64 EXCEPTION_ACCESS_VIOLATION (null pointer)
- Viewport content bleeding/flickering into other editor windows
- Crash when opening floating windows (Concert browser, Blueprint editor, Material editor, Preferences)
- Black viewport with no lighting

## Root Cause
NVIDIA's D3D12 presentation layer crashes when UE5 creates multiple swap chains (one per floating window). RTX 5090 drivers are still maturing for UE5's DX12 pipeline.

## CRITICAL: Never switch between Studio and Game Ready drivers
Switching without DDU leaves mismatched DLLs that cause persistent flickering/crashes. If you must switch, use DDU in Safe Mode first.

## Config fixes (DefaultEngine.ini)
```ini
[/Script/Engine.RendererSettings]
r.RHICmd.ParallelTranslate.Enable=0
r.Shadow.Denoiser=0
D3D12.ResidencyManagement=0
r.RayTracing.PersistentSBT=0
```
These keep ray tracing ON but disable known crash triggers.

## Other fixes
- Disable Hardware-Accelerated GPU Scheduling (Windows Settings > Display > Graphics)
- Disable Fast Startup (Control Panel > Power Options)
- Increase TDR timeout: HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\GraphicsDrivers\TdrDelay = 60
- Dock all editor windows (don't use floating windows)
- Close floating windows before joining Concert

## Driver versions
- 591.74 — reported to fix NvPresent64 for some users
- 576.80 — resolved RTX 50-series crashes from 576.02
- 577.00 — last stable before DoF regressions in 580+
- Avoid 580.xx and 581.xx branches
